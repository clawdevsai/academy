10. Mantendo contexto entre tarefas e reduzindo desperdício de tokens

A janela de contexto é o recurso mais caro e mais escasso de um agente — aprenda a geri-la como um orçamento.

🎯 Objetivos de aprendizagem

  • Entender por que contexto não é "memória infinita" e o que acontece quando ele se esgota.
  • Usar subagentes para isolar trabalho "sujo" (logs, buscas grandes, exploração) do contexto principal.
  • Substituir repetição de contexto por arquivos de referência persistentes (CLAUDE.md, memória).
  • Saber quando dividir uma tarefa grande em etapas menores e quando resetar o contexto.
  • Aplicar técnicas práticas de compactação e resumo para conversas longas.

Explicação conceitual

Todo modelo de linguagem opera dentro de uma janela de contexto: um limite fixo de tokens que inclui tudo que já foi dito na conversa, o conteúdo de arquivos lidos, resultados de ferramentas, instruções do sistema e a resposta que está sendo gerada. Diferente de uma memória de computador tradicional, essa janela não é "apagada e reescrita" de forma barata — cada token que entra nela custa processamento, custa dinheiro (em APIs pagas por token) e, o mais importante para você como praticante, compete por atenção com o resto da conversa.

Um agente com contexto poluído — cheio de logs de erro extensos, arquivos inteiros colados sem necessidade, ou histórico de tentativas fracassadas — não fica apenas mais lento e mais caro. Ele fica pior: a informação relevante para a tarefa atual se dilui em meio a ruído, e o modelo passa a "esquecer" instruções dadas no início da conversa (o efeito é conhecido informalmente como "lost in the middle", onde informação no meio de contextos longos recebe menos atenção do que informação no início ou no fim).

A boa notícia é que o Claude Code foi desenhado com esse problema em mente. As mesmas ferramentas que você já conhece dos capítulos anteriores — subagentes, arquivos de memória, Spec-Driven Development — são, na prática, técnicas de gestão de contexto. Neste capítulo vamos olhar para elas especificamente sob essa lente: não "como fazer o agente trabalhar", mas "como manter a janela de contexto enxuta e relevante ao longo de uma sessão longa ou de um projeto inteiro".

Exemplos práticos

Imagine que você precisa investigar por que um endpoint de API está retornando erro 500 intermitente em produção. Os logs têm milhares de linhas. Trazer esse arquivo inteiro para o contexto principal seria desperdício — a maior parte das linhas é ruído. O jeito certo é delegar a um subagente: 

Você: Tenho um arquivo de log de 40MB em /var/log/api/error.log.
Preciso saber por que o endpoint POST /orders está retornando 500
de forma intermitente nas últimas 24h. Use um subagente para
vasculhar o log e me traga só um resumo com: padrão do erro,
horários, e stack trace relevante (se houver repetição).

O subagente lê o arquivo inteiro, filtra, cruza informações — e tudo isso acontece na janela de contexto dele, não na sua. O que volta para a conversa principal é um resumo de algumas linhas. Essa é a diferença entre gastar 40MB de tokens e gastar algumas centenas.

Outro exemplo comum: em vez de colar um arquivo de 800 linhas para pedir uma alteração em uma função específica, peça ao agente para localizar e ler apenas o trecho relevante: 

Ruim:  "Aqui está o arquivo inteiro [cola 800 linhas], corrija a função calcularFrete"
Melhor: "No arquivo src/services/frete.js, revise a função calcularFrete.
         Leia o arquivo você mesmo e aplique a correção X."

O agente tem ferramentas de leitura de arquivo — deixe-o buscar exatamente o que precisa, em vez de você pré-carregar tudo "por garantia".

Fluxo de trabalho recomendado

  1. No início do projeto, escreva um CLAUDE.md com convenções, arquitetura, comandos de build/teste. Isso substitui repetir esse contexto em cada prompt.
  2. Antes de uma tarefa grande, pergunte-se: "isso precisa de exploração ampla (logs, busca em muitos arquivos)?" Se sim, delegue essa parte a um subagente.
  3. Divida tarefas grandes em fases com checkpoints — planejar, implementar, revisar, testar — em vez de pedir tudo em um único prompt monstro.
  4. Ao trocar de assunto (nova feature não relacionada), inicie uma nova conversa/sessão em vez de continuar acumulando contexto antigo.
  5. Em conversas longas que precisam continuar, use os recursos de compactação/resumo disponíveis (por exemplo, pedir explicitamente um resumo do progresso antes de seguir, ou usar o comando de compactação do Claude Code quando disponível).

Dicas e boas práticas

Dica: Trate o CLAUDE.md como a "memória de longo prazo" do projeto. Qualquer informação que você se pega repetindo em vários prompts é candidata a virar uma linha nesse arquivo.
Dica: Prefira "leia o arquivo X e faça Y" a colar o conteúdo manualmente. O agente sabe usar suas próprias ferramentas de leitura de forma mais econômica do que você acha.
Dica: Se uma sessão está ficando longa e você percebe respostas mais lentas ou genéricas, é sinal de que o contexto está saturado — é hora de resumir ou começar de novo.

Erros comuns

Erro comum: Colar arquivos inteiros "só para garantir que o agente veja tudo". Isso quase sempre é desperdício — o agente tem ferramentas para buscar o que precisa sob demanda.
Erro comum: Manter a mesma conversa aberta por dias, acumulando contexto de tarefas completamente diferentes. Isso degrada a qualidade das respostas e aumenta custo sem benefício.
Erro comum: Não usar subagentes para exploração e deixar o agente principal "ler tudo" diretamente — poluindo o contexto que deveria ficar reservado para a decisão e a implementação.

Exercício prático

Mão na massa: Escolha um projeto real seu com pelo menos um arquivo de log ou um diretório com muitos arquivos de código. Peça ao Claude Code para investigar algo usando explicitamente um subagente (ex.: "use um subagente para mapear todos os lugares onde a função X é chamada e me traga só a lista de arquivos e linhas"). Compare o tamanho da resposta final com o volume de dados que foi processado — essa é a economia de contexto na prática.

Resumo do capítulo

Contexto é um recurso finito e caro, não uma memória infinita. Subagentes permitem isolar trabalho volumoso (logs, buscas amplas) do contexto principal, devolvendo apenas o resumo relevante. Arquivos de referência como o CLAUDE.md eliminam a necessidade de repetir instruções a cada prompt. Tarefas grandes devem ser divididas em etapas, e conversas não relacionadas devem começar do zero em vez de acumular ruído. Gerenciar contexto bem é, na prática, gerenciar a qualidade e o custo de tudo que o agente produz.
