2. O que é o Claude Code e como ele funciona

Conhecendo a ferramenta de linha de comando da Anthropic para desenvolvimento agentic.

🎯 Objetivos de aprendizagem

  • Entender o que é o Claude Code e em que ele difere de um assistente de chat comum.
  • Compreender o "loop de tarefas" (task loop) que faz o agente ler, agir e verificar seu próprio trabalho.
  • Conhecer as capacidades básicas: leitura e edição de arquivos, execução de comandos de shell, testes e git.
  • Saber instalar e rodar o Claude Code num projeto real pela primeira vez.

Explicação conceitual

O Claude Code é uma ferramenta de linha de comando (CLI, do inglês command-line interface — um programa que você usa digitando comandos no terminal, em vez de clicar em botões) criada pela Anthropic para desenvolvimento agentic. Isso significa que, diferente de um chat onde você copia e cola código, o Claude Code roda dentro (ou ao lado) do seu projeto e tem permissão para agir diretamente sobre ele: ler arquivos do repositório, editá-los, criar novos arquivos, rodar comandos de terminal, executar testes e interagir com o git.

A diferença central em relação a um chat comum é que o Claude Code opera em um loop de tarefas: ele não apenas responde uma vez e para. Ele lê o que precisa, toma uma ação, observa o resultado dessa ação (por exemplo, o output de um teste que rodou), decide o próximo passo com base nisso, e repete esse ciclo até considerar a tarefa concluída ou até precisar da sua confirmação para continuar.

flowchart TD A[Você dá uma instrução] --> B[Claude Code lê arquivos relevantes do projeto] B --> C[Planeja os passos necessários] C --> D[Executa uma ação: edita arquivo, roda comando, etc.] D --> E{Resultado esperado?} E -- Não --> C E -- Sim --> F{Tarefa completa?} F -- Não --> C F -- Sim --> G[Mostra resumo do que foi feito]

Esse loop é o que torna o Claude Code "agentic" e não apenas "conversacional". Ele pode, por exemplo, escrever uma função, rodar os testes automaticamente, ver que um teste falhou, ler a mensagem de erro, corrigir o código e rodar os testes de novo — tudo isso sem que você precise intervir a cada passo.

Para isso funcionar, o Claude Code precisa de acesso a algumas capacidades básicas, chamadas de tools (ferramentas) — vamos aprofundar esse conceito no Capítulo 4, mas por ora basta saber que existem ferramentas nativas como:

FerramentaO que faz
Leitura de arquivosAbre e lê o conteúdo de arquivos do projeto para entender o código existente.
Edição de arquivosFaz alterações precisas em arquivos existentes ou cria novos.
Execução de shellRoda comandos de terminal: instalar dependências, rodar scripts, compilar.
TestesExecuta a suíte de testes do projeto e interpreta os resultados.
GitCria commits, branches, vê diffs e histórico de mudanças.
Busca no repositórioProcura por trechos de código, funções ou padrões em todo o projeto.

Outro ponto importante: o Claude Code é autônomo, mas não descontrolado. Ele pede confirmação antes de ações potencialmente arriscadas (como rodar um comando que apaga arquivos) e você pode configurar o nível de autonomia — desde "pergunte antes de cada ação" até "execute a tarefa inteira e só me avise no final". Essa configuração de permissões é essencial para usar a ferramenta com segurança, especialmente em projetos de produção.

1 e 2 — Instalar e logar no Claude Code (obrigatório)

# instala o Claude Code globalmente (requer Node.js instalado)
npm install -g @anthropic-ai/claude-code

# entra na pasta do projeto (ex.: user-api, nossa API de exemplo)
cd user-api

# inicia o Claude Code — na primeira execução ele pede login
# (abre o navegador para autenticar com sua conta Anthropic/Claude)
claude

Sem login concluído, o Claude Code não executa nenhuma tarefa — é o único passo verdadeiramente bloqueante antes de começar.

3 a 6 — Plugins opcionais de produtividade

context-mode, ponytail, caveman e karpathy-guidelines são plugins/skills de terceiros que mudam como o Claude Code processa contexto, estilo de resposta e disciplina de engenharia (vamos usá-los ao longo do curso, mas eles não são exigidos para acompanhar o conteúdo). Cada repositório documenta seu próprio método de instalação, mas o padrão do Claude Code para plugins de marketplace é:

> /plugin marketplace add <owner>/<repo>
> /plugin install <nome-do-plugin>
Dica: antes de instalar, confira o README de cada repositório — o nome exato do plugin no marketplace e eventuais passos extras (variáveis de ambiente, dependências) podem mudar entre versões.

7 — Spec Kit (obrigatório)

Usado a partir do Capítulo 8 (Spec-Driven Development) e por vários dos agentes personalizados do curso. Instale a CLI specify conforme o README do repositório e inicialize a integração com Claude Code dentro do projeto user-api:

cd user-api
specify init . --integration claude

Isso cria a estrutura .specify/ e as skills speckit-* (specify, clarify, plan, tasks, analyze) usadas nos capítulos sobre spec-driven development e nos agentes de arquitetura/implementação.

8 — skill-creator (opcional)

Plugin oficial do marketplace claude-plugins-official para criar novas skills seguindo a estrutura correta (frontmatter YAML, progressive disclosure). Útil se você quiser construir skills próprias além das usadas no curso, mas não é exigido para acompanhar o conteúdo.

claude plugin install skill-creator@claude-plugins-official
claude plugin install skill-creator@claude-plugins-official

Exemplos práticos

Com o Claude Code instalado, logado e o Spec Kit inicializado, rodar o primeiro comando dentro do projeto user-api:

cd user-api
claude

Depois de iniciado, você conversa com ele em linguagem natural, no próprio terminal:

> Leia o pyproject.toml e me diga quais dependências
  desatualizadas o projeto tem hoje.

Ou peça uma tarefa mais completa, que exige várias ações em sequência:

> Crie o endpoint POST /usuarios que valide nome e e-mail,
  persista o usuário usando o repositório em src/adapters/db,
  e escreva um teste em tests/ para o caso de e-mail inválido.

Nesse segundo exemplo, o Claude Code provavelmente vai: (1) explorar a pasta src/adapters/db e o domínio em src/domain para entender o padrão hexagonal usado no projeto, (2) criar o endpoint seguindo esse padrão, (3) escrever o teste em tests/, e (4) rodar uv run pytest para confirmar que tudo passa — tudo isso de forma autônoma, mostrando o progresso no terminal.

Fluxo de trabalho recomendado

  1. Abra o terminal dentro da pasta do projeto (não fora dela) — o Claude Code usa o diretório atual como ponto de partida para entender o repositório.
  2. Comece com um pedido claro e delimitado, especialmente nas primeiras vezes.
  3. Acompanhe o que ele está fazendo: qual arquivo está lendo, qual comando está rodando.
  4. Quando ele pedir confirmação para uma ação sensível, leia antes de aprovar.
  5. Ao final, revise o diff (as mudanças) antes de fazer commit.

Dicas e boas práticas

Dica: use um arquivo CLAUDE.md na raiz do projeto para descrever convenções, comandos de build/test e preferências do time — no user-api, por exemplo, documentando que os testes rodam com uv run pytest, o lint com ruff check e que a arquitetura é hexagonal. O Claude Code lê esse arquivo automaticamente e passa a seguir essas regras em todas as tarefas.
Dica: em projetos críticos, comece com o modo de permissões mais conservador (confirmação a cada ação) e vá relaxando conforme ganha confiança na ferramenta.
Dica: use controle de versão (git) sempre. Isso te dá uma rede de segurança: se o resultado não for o esperado, é fácil reverter.

Erros comuns

Erro comum: rodar o Claude Code fora da pasta do projeto ou numa pasta errada, fazendo com que ele não encontre o contexto certo. Sempre confirme em qual diretório você está antes de iniciar.
Erro comum: dar autonomia total (aprovar tudo sem revisar) logo no início, antes de entender como a ferramenta se comporta no seu projeto específico. Ganhe confiança gradualmente.
Erro comum: pedir tarefas gigantes e vagas de uma vez ("refatore o sistema inteiro"). Isso deixa a IA sem critério claro de sucesso e aumenta a chance de erro. Prefira tarefas menores e mais bem definidas, como vimos no Capítulo 1.

Exercício prático

Mão na massa: instale o Claude Code na pasta do user-api (ou de um projeto de estudo equivalente). Peça para ele explicar a estrutura de pastas do projeto em português (domínio em src/domain, adapters em src/adapters, testes em tests), e depois peça uma pequena mudança (ex.: adicionar um comentário explicativo no caso de uso de cadastro de usuário, ou criar um README.md básico). Observe no terminal quais ferramentas ele usou (leitura, escrita, etc.) durante a execução.

Resumo do capítulo

O Claude Code é uma CLI da Anthropic para desenvolvimento agentic: ele lê, edita e testa código dentro do seu projeto de forma autônoma, operando em um loop de tarefas até concluir o objetivo pedido. Diferente de um chat, ele age diretamente sobre o repositório usando ferramentas como leitura/edição de arquivos, shell, testes e git. A autonomia é configurável, e cabe a você calibrar o nível de supervisão conforme a confiança que ganha na ferramenta e a criticidade do projeto.
