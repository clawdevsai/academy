1. Introdução ao Desenvolvimento de Software com IA

Como a inteligência artificial mudou a forma de programar — e por que isso importa para você, mesmo sem experiência prévia.

🎯 Objetivos de aprendizagem

  • Entender a diferença entre "usar IA para programar" e "desenvolvimento agentic" (baseado em agentes autônomos).
  • Conhecer a evolução: autocomplete → chat → agentes que executam tarefas sozinhos.
  • Compreender os conceitos-chave que vão aparecer no resto do curso: LLM, prompt, contexto, agente.
  • Identificar em quais situações vale a pena usar IA no seu fluxo de desenvolvimento.

Explicação conceitual

Se você nunca programou com ajuda de inteligência artificial, a primeira coisa a entender é que "IA no desenvolvimento" não é uma coisa só. É um espectro, e a maioria das pessoas conhece só a ponta mais simples dele: o autocomplete inteligente. Este capítulo existe para te mostrar o mapa completo antes de mergulharmos no Claude Code nos próximos capítulos.

Um LLM (Large Language Model, ou "modelo de linguagem grande") é o motor por trás de tudo isso: um sistema treinado com uma quantidade enorme de texto — incluindo código de milhões de repositórios públicos — que aprendeu a prever qual é a próxima palavra (ou "token", um pedacinho de texto) mais provável dado um contexto. Isso parece simples, mas na prática significa que o modelo consegue escrever código, explicar conceitos, encontrar bugs e sugerir arquiteturas, porque ele "viu" padrões parecidos milhares de vezes durante o treinamento.

O que mudou nos últimos anos não foi só o modelo ficar mais inteligente. Mudou a forma como ele interage com o seu código. Podemos pensar nisso em três gerações:

GeraçãoComo funcionaExemploLimitação principal
1. AutocompleteSugere a próxima linha ou função enquanto você digita, dentro do editor.GitHub Copilot (modo inline)Não entende o projeto inteiro, só o arquivo aberto.
2. Chat / assistenteVocê copia e cola código numa conversa, pede explicação ou sugestão, e cola a resposta de volta.ChatGPT, Claude.aiVocê é o "cola-tudo": o modelo não vê o projeto nem executa nada.
3. Agentic (agente autônomo)A IA lê o repositório sozinha, decide o que fazer, edita arquivos, roda comandos, testa e itera até o resultado ficar pronto.Claude CodeExige supervisão e bom uso de instruções — não é "mágica sem controle".

Este curso é sobre a terceira geração. É a diferença entre pedir uma receita de bolo por escrito e ter alguém que vai até a cozinha, separa os ingredientes, assa o bolo e te chama quando estiver pronto — perguntando antes se tiver dúvida.

Dois termos vão aparecer o tempo todo daqui para frente, então vale fixar agora:

  • Prompt: a instrução em linguagem natural que você dá para a IA. Pode ser uma frase curta ("corrija esse bug") ou um parágrafo detalhado com contexto, restrições e exemplos.
  • Contexto: tudo aquilo que o modelo "enxerga" no momento de responder — o histórico da conversa, arquivos lidos, resultado de comandos executados. Quanto melhor o contexto, melhor a resposta. Gerenciar contexto bem é uma habilidade central quando se trabalha com agentes.

Exemplos práticos

Para deixar concreto, veja a mesma tarefa — "adicionar validação de e-mail num formulário" — resolvida nas três gerações:

Imagine que você está no projeto user-api (uma API em Python/FastAPI) e precisa validar o e-mail no endpoint de cadastro de usuário. Veja a mesma tarefa — "validar e-mail no cadastro de usuário" — resolvida nas três gerações:

Geração 1 (autocomplete): você começa a digitar def validar_email( em src/domain/use_cases/cadastrar_usuario.py e o editor sugere o corpo da função. Você aceita, mas ainda precisa descobrir sozinho onde chamar essa função, tratar o erro na camada HTTP e testar.

Geração 2 (chat): você abre o Claude.ai, cola o código do endpoint POST /usuarios e pergunta "como valido o e-mail aqui?". Recebe uma resposta com código, mas precisa copiar manualmente, colar no lugar certo, adaptar nomes de variáveis e testar.

Geração 3 (agentic, com Claude Code): você escreve um prompt assim, no terminal do projeto user-api:

Adicione validação de e-mail no caso de uso de cadastro de usuário,
em src/domain/use_cases/cadastrar_usuario.py.
Se o e-mail for inválido, o endpoint POST /usuarios deve retornar 422
com uma mensagem clara.
Rode "uv run pytest" depois para garantir que nada quebrou.

O agente localiza o arquivo sozinho, lê o código ao redor para entender o padrão usado no projeto (arquitetura hexagonal, com domínio separado dos adapters HTTP), escreve a validação, edita o arquivo diretamente, roda a suíte de testes com uv run pytest e te mostra o resultado — tudo sem você precisar copiar e colar nada.

Fluxo de trabalho recomendado

Antes de instalar qualquer ferramenta, vale internalizar o papel que você vai ter neste novo modelo de trabalho. Você deixa de ser quem digita cada linha e passa a ser quem define objetivos, dá contexto e revisa o resultado.

sequenceDiagram participant Você participant Agente de IA participant Repositório Você->>Agente de IA: Descreve o objetivo (prompt) Agente de IA->>Repositório: Lê arquivos relevantes Agente de IA->>Agente de IA: Planeja os passos Agente de IA->>Repositório: Edita código, roda comandos Agente de IA->>Você: Mostra o resultado e pede revisão Você->>Agente de IA: Aprova ou pede ajustes
  1. Defina o objetivo com clareza (o que muda, onde, e qual o critério de "pronto").
  2. Dê contexto suficiente (nomes de arquivos, padrões do projeto, restrições).
  3. Deixe o agente trabalhar e observe o que ele está fazendo.
  4. Revise o resultado antes de aceitar — você continua responsável pelo código.
  5. Refine o prompt se o resultado não for o esperado, em vez de corrigir manualmente tudo.

Dicas e boas práticas

Dica: comece com tarefas pequenas e bem definidas enquanto aprende a "conversar" com um agente. Isso cria confiança antes de delegar tarefas maiores.
Dica: pense em prompt como briefing de trabalho, não como busca no Google. Quanto mais contexto relevante você der, melhor o resultado — assim como orientar um colega novo no time.
Dica: você não precisa saber programar para começar a entender esse curso, mas noções básicas de lógica e de como um projeto de software é organizado (pastas, arquivos, versionamento) vão acelerar seu aprendizado.

Erros comuns

Erro comum: achar que "geração 3" significa não precisar mais entender o que o código faz. Você ainda precisa revisar e validar — a IA erra, principalmente em tarefas ambíguas ou mal especificadas.
Erro comum: tratar um agente como um mecanismo de busca ("me diz a resposta") em vez de um executor de tarefas ("faça essa mudança"). São usos diferentes e pedem prompts diferentes.
Erro comum: dar instruções vagas ("melhore esse código") e se frustrar com o resultado. Sem critério de sucesso claro, o agente vai adivinhar o que você quer — e pode adivinhar errado.

Exercício prático

Mão na massa: pegue uma tarefa simples do projeto user-api (ex.: validar o nome do usuário no cadastro, impedir e-mail duplicado, formatar a resposta de erro do endpoint POST /usuarios). Escreva três versões do mesmo pedido: uma como se fosse para o autocomplete (uma linha de código incompleta, tipo def validar_nome(), uma como se fosse para um chat (uma pergunta sobre o código do endpoint), e uma como se fosse para um agente (um objetivo completo, com o caminho do arquivo, contexto e critério de sucesso, como "rodar uv run pytest ao final"). Compare as três e observe como a forma de pedir muda o nível de detalhe necessário.

Resumo do capítulo

O desenvolvimento de software com IA evoluiu em três gerações: autocomplete, chat e agentic. Este curso foca na terceira, onde ferramentas como o Claude Code leem o repositório, executam tarefas e iteram sozinhas até entregar um resultado, sob sua supervisão. Os conceitos de LLM, prompt e contexto são a base de tudo que vem a seguir. O seu papel muda de "quem escreve cada linha" para "quem define objetivos e revisa resultados" — uma habilidade que vamos desenvolver ao longo do curso.
